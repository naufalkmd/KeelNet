# KeelNet Stage Note Template

Use this for any stage after Stage 1 when you want a short, shareable run note.

Keep it practical. The goal is to make the next run easier, not to write polished documentation.

Recommended update points:

1. before implementation, fill in the goal, success condition, and planned commands
2. after smoke test, record environment issues and command fixes
3. after a meaningful run, fill in metrics, verdict, and next actions
