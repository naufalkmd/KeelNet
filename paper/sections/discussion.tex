The current project deliberately favors scope control over breadth.
That is a strength for the first paper draft, but it also creates limitations
that should be stated clearly.

\paragraph{Current limitations.}
\begin{itemize}[leftmargin=1.25em]
  \item The active implementation is limited to Stage 1.
  \item The task uses a single provided evidence passage rather than retrieved
  evidence.
  \item The action space is intentionally narrow and does not yet cover richer
  support labels or multi-hop settings.
  \item The current results will reflect one controlled dataset setup rather
  than a broad claim about hallucination in general.
\end{itemize}

\paragraph{How to interpret a positive result.}
If Stage 1 shows that unsupported answers fall while answerable performance
remains reasonably stable, then the main conclusion should be narrow but useful:
explicit abstention supervision and abstention-aware evaluation can improve the
answer-versus-abstain trade-off even in a simple extractive QA system.
That would not explain hallucination in general, but it would support the idea
that aligned targets and aligned metrics matter.

\paragraph{How to interpret a mixed or negative result.}
If unsupported answers decrease only when answerable performance collapses, or
if gains appear only at brittle threshold settings, then Stage 1 should be read
as evidence that abstention supervision alone is insufficient.
That outcome would still be valuable because it would justify moving attention
toward support verification, calibration, or stronger control mechanisms rather
than over-claiming what a thresholded extractive model can do.

\paragraph{Why this stage is still useful.}
Even under its narrow scope, Stage 1 tests one defensible claim with clean
supervision and interpretable failure modes.
If Stage 1 cannot show a meaningful trade-off improvement, that is important
negative evidence before investing in more complex later stages.

\paragraph{Next paper-facing extensions.}
Once Stage 1 is stable, the most natural paper extensions are:
\begin{itemize}[leftmargin=1.25em]
  \item adding support verification,
  \item calibrating confidence against correctness and support,
  \item testing retrieval noise, and
  \item comparing adaptive balancing against simpler fixed baselines.
\end{itemize}
